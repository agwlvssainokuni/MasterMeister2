# API ドキュメント

## REST API

**現状は存在しない。** `backend/src` 配下には `@RestController` / `@Controller` クラスは1つもない。`spring-boot-starter-web` はクラスパス上にあるが、自動設定以外では未使用。

`docs/REQUIREMENTS.md` によれば、以下のREST APIが計画されている（エンドポイントレベルの設計はまだ行われていない）:
- ユーザ登録（メールアドレス送信、トークンによるパスワード設定完了）と管理者による承認/却下
- 対象RDBMS接続設定とスキーマ取り込み
- テーブル/カラム権限設定（YAMLインポート/エクスポート付き）
- ログイン/セッション（またはJWT）認証
- マスタデータの一覧/絞込/編集 — **作成・更新・削除のすべての操作を単一のトランザクションAPIエンドポイントに統合**（意図的な設計、見落としではない）
- クエリビルダーのSQL生成/逆解析
- 保存クエリのCRUD＋実行
- 都度入力クエリの実行（読み取り専用、パラメータ化 `:param` 対応、任意ページング）
- クエリ実行履歴の一覧/絞込
- 監査ログ閲覧（管理者のみ）

## 内部API

**現状は存在しない。** 生成された `MasterMeisterApplication` エントリポイント以外に、サービスインタフェース、リポジトリインタフェース、その他の内部契約は一切書かれていない。

## データモデル

**現状は存在しない。** JPAエンティティ、DTO、`NamedParameterJdbcTemplate` にマッピングされる行クラスは一切存在しない。`docs/PROJECT_STRUCTURE.md` によれば、計画中のエンティティ領域はユーザ、RDBMS接続設定、取り込んだスキーマメタデータ、テーブル/カラム権限、保存クエリ、クエリ実行履歴、監査ログエントリであり、すべて内部H2データベースにJPA経由で格納される予定。
